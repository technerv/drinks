AN INTERACTIVE GIS WEB APPLICATION TOOL FOR MANAGEMENT OF ALCOHOL SELLING ENTITIES

Background of the study

Eldoret city has a cosmopolitan and urban population of approximately 289,380. With a steady rise in corresponding economic growth, it has led to the explosion of population in the cityâ€™s residential estates and the proliferation of many bars, wines and spirits and illicit brew dens to cater for the different economic capabilities of consumers.


Background of the study

Because of this, access to alcohol has been made easy even to school going children and idle jobless youth. The presence of alcohol joints near roads also poses risks to road users. The easy access means that matatu drivers and bodaboda drivers can take alcohol posing risks.

Background of the study

This study undertakes to map visualize and categorize all alcohol joints within our residential estates. It also seeks to establish whether these places are observing the law.


Problem Statement

  • Alcoholism has permeated all classes of human society, from the lowly paid to the rich.
  • To make matters worse, alcohol selling business has moved from towns to estates. As a consequence, Kenyans are taking alcohol since it is readily available in estates.
  • Kids also get exposed to alcohol at an early age.


Main objective

Collect GPS data and map all legal and illegal alcohol joints within Eldoret estates of Maili Nne, Huruma, Road Block, Mwanzo, Langas, Action, Annex.

Specific Objectives

  • Produce soft and hard copy maps showing location of alcohol selling entities.
  • Determine the concentration of alcohol selling entities given the population density per square kilometer.

Limitations

  • Collection of daily supply and consumption data from the business entities is hampered by lack of appropriate law which would have been critical in alcohol consumption prediction modeling.
  • Hostile reception by illicit dens will hamper the collection of gps locations.

Limitations

  • Accurate and updated population data is unavailable, since the last census was held 8 years ago. This data is critical to determine the maximum allowed alcohol selling entities given for a unit of population density per square kilometer.
  • The collection GPS locations depend on smartphones or specialized gps devices which are expensive, and which require an internet connection to submit data to the server hosting the application.

Benefits

  • Since it is a web application, it can be accessed anywhere, anytime.
  • The system is cost effective since it is built on free and open source software.
  • The availability of the tool can provide a framework which other applications can be built upon.
  • GIS web applications support real time spatial analysis of data.
  • Compliance reports can be collected by county government based on data collected.

Methodology

  • The Interactive GIS Web Application Tool will adopt the Iterative model, where software is built in small chunks, each time adding more functions to it. You can implement a small set of software requirements, then test, evaluate and refine the requirements.
  • With each iteration, new requirements are added and a new version of the software is produced. This process is repeated until the application is fully developed and all requirements implemented.

Methodology

  • One advantage of Iterative model is that we get a working version of the application early in the process and so it less expensive to implement changes.

Hardware Requirements

  • Intel Core i3 computer or above with 500GB hard disk, 4GB RAM, DVD ROM.
  • Garmin GPS Tracker for marking and tracking cordinates and waypoints.

Software requirements

  • Ubuntu Linux 16.04 operating system
  • QGis
  • Pip
  • Django
  • Java Open Street Map Editor
  • PyCharm (Python Editor and Console)
  • GeoDjango Module Framework
  • PostgreSQL for storing database information

Functional requirements

  • Monitoring module: The availability of an interactive web visualization tool will enable agencies to monitor and analyze the proliferation of alcohol joints.
  • Administrator Module: This is the backend administration where different users will be assigned to manage and manipulate data that is viewed on the map.

Functional requirements

  • Reports Module: Produce soft and hard copy maps showing location of alcohol selling entities.
  • Location View Module: This tool will enable agencies to monitor and analyze the proliferation of alcohol joints. Those agencies will know the exact locations of these alcohol joints, so that in case of breach of law, they can be punished. It will also enable help provide legal locations during provision of permits and licenses to new business owners.
